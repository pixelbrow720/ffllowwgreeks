Apa itu FlowGreeks?

Real-time options flow + dealer positioning intelligence

Platform yang nge-translate state buku dealer jadi sinyal aktionable buat trader 0DTE SPX & NDX. Bukan analytics terminal — tapi "apa yang HARUS dilakukan dealer berikutnya" dalam dollar notional.

Tagline: "Read the Dealer."

Distribusi: add-on di dalam flowjob.id (parent product). Tidak dijual standalone — akses via opaque API key dari bootcamp graduation.

Differentiator

  • Predictive forced-flow — bukan cuma show GEX, tapi simulasi paksa hedging dealer
  • Charm Clock — visualisasi window decay charm intraday
  • 0DTE-only — no 3,500-ticker bloat, deliberate narrow surface
  • Replay + Backtest — time machine + signal validation
  • Desktop-only — terminal-grade, no mobile compromise

DPI — Dealer Pressure Index

0–100 EWMA composite, 5 komponen tertimbang

NetGEX_pressure   = clip01(|NetGEX|/5e9) × 100
                       weight: 0.30

Charm_velocity    = clip01(|Δcharm|/5e6) × 100
                       weight: 0.25

Vanna_sensitivity = clip01(|vanna|/1e6) × 100
                       weight: 0.20

TTC               = (timeLeft/sessionLen)^1.5 × 100
                       weight: 0.15   convex ramp ke EOD

Flow_concentration= clip01(HHI × 5.0) × 100
                       weight: 0.10   HHI = Σ(flow_i/Σflow)²
─────────────────────────────────────
Σ smoothed (EWMA α=0.3) → DPI 0–100

Intraday timing engine

Charm Clock — 5 zona

velocity (Δdelta/min)
   │              ┌──── PEAK ────┐
   │     RISING ──┘              └── FADING
   │   ╱                                ╲
   │ ╱                                    ╲ PIN
   │  WEAK                                  ▔▔

   09:30                                  16:00 ET

Klasifikasi zona berdasarkan |charm velocity| evolution dalam session. Two-generation reset (deep-review fix) cegah memori basi dari sesi sebelumnya bocor ke pagi berikutnya.

Pin Probability Engine

EOD candidate strike i:

  weight_i = OI_i × |gamma_i|
  proximity_i = exp(-((spot - K_i)/σ)²)

  P(pin = K_i) = weight_i × proximity_i
                 ─────────────────────
                 Σⱼ weight_j × prox_j

Output: probability mass function across active strikes. Yang dirender ke FE adalah top-N strike + spot offset.

What-If Simulator

Reprice setiap kontrak di shocked spot/time/vol, re-aggregate dealer net delta & gamma, return forced-hedge notional dalam $.

Auth Model

Pivot: dari JWT/account → opaque API key

FlowGreeks bukan stand-alone product. Akses cuma di-grant ke alumni bootcamp flowjob.id via API key yg di-mint parent site.

Yang dihapus dari backend ini (~1500 LOC):

  • signup / login / password
  • JWT issuer + refresh rotation
  • account lockout
  • tier gating

Semua itu dipindah ke flowjob.id. Binary FlowGreeks cuma authenticate inbound traffic vs hashed key di table api_keys (shared Postgres).

5-layer defense-in-depth

  1. API-key middleware
    Bearer / X-API-Key, hash compare
  2. Per-key rate limit
    rate_limit_rps + rate_burst dibaca langsung dari row api_keys → flowjob.id bisa hot-swap tier tanpa redeploy
  3. HTTP body cap + WS read limits
  4. Security response headers
    nosniff · DENY-frame · CSP default-src 'none' · CORP same-origin · HSTS gated TLS
  5. Audit log + Prometheus metrics + alert rules

Time Machine

Replay + Backtest engine

Replay (cmd/replay)

Swap-in pengganti cmd/ingest. Baca historical dari Postgres, paced ulang, publish ke NATS subject yang sama. Downstream (compute, api, FE) nggak tau bedanya.

  • Pacing: 1× realtime · N× fast-forward · 0× unpaced (CPU-bound)
  • Control plane: /ws/replay/{session_id}
  • Pause / resume / seek / speed dari FE

Backtest (POST /api/backtest/run)

Predicate-driven strategy validation langsung di atas archive dealer_state_1s.

input:  alerts.Rule (Kind+Threshold+StringArg)
        + window [from, to]
        + symbol

engine: walk dealer_state_1s rows
        evaluate predicate per row
        synthesize trade entries/exits
        compute P&L curve

output: Sharpe · Sortino · maxDD
        · trade list · equity curve
        deadline: 30s

Re-use yang sama predicate dari alerts engine — saved alert langsung backtest-able tanpa redefinition.
