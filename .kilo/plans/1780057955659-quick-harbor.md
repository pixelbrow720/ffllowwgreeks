# Plan: Create root `AGENTS.md` for FlowGreeks workspace

## Goal

Create a single, compact `C:\FLOWGREEKS\AGENTS.md` that gives any non‑Claude agent
(OpenCode, Kilo, Cursor, Codex, etc.) the high‑signal repo‑specific facts they
will otherwise guess wrong. Keep it short. Defer to `CLAUDE.md` and `HANDOFF.md`
for the deep rules and current session state instead of duplicating them.

No `AGENTS.md` exists yet at the workspace root. This is a fresh creation.

## Why a separate `AGENTS.md` (not just CLAUDE.md)

- Kilo's project config explicitly looks for `AGENTS.md` (see env in system prompt).
- OpenCode / Codex / Cursor / Aider all converge on `AGENTS.md` as the cross‑agent
  standard.
- `CLAUDE.md` is detailed, opinionated, and tuned to Claude Code workflows
  (entry‑point reading order, `/init` ritual, B. Indonesia tone). It should
  stay as is.
- `AGENTS.md` should be the smallest possible file that prevents the most common
  agent mistakes, then point at `CLAUDE.md` + `HANDOFF.md` for the rest.

Treat both as authoritative. `AGENTS.md` is the **lobby**, `CLAUDE.md` /
`HANDOFF.md` are the **rooms**.

### 5. Architecture an agent won't infer from filenames

Pipeline (Go binaries glued by NATS JetStream):

```
OPRA / GLBX  →  cmd/ingest  →  NATS  →  cmd/compute  →  Postgres + Redis
                                                            ↓
                                                         cmd/api  →  REST + /ws/live  →  web/
```

`cmd/replay` swaps in for `cmd/ingest` for historical sessions; everything
downstream is unchanged. Hot‑path packages live in `backend/internal/`:

- `feed/` OPRA Pillar + GLBX MDP3 parsers (Databento `dbn-go`)
- `greeks/` Black‑Scholes + IV (Brent) + analytic Greeks
- `dealer/` GEX, DPI 5‑component, Charm Clock, Pin engine, forced‑flow simulator
- `bus/` NATS pub/sub
- `store/` TimescaleDB hypertables (`ticks`, `dealer_state_1s`) + Redis SWC
- `api/` chi router, REST, `/ws/live`, `/ws/replay`
- `apikey/` opaque API‑key auth (parent product mints rows)

Single contract source of truth for TS types: `backend/docs/openapi.yaml`.
Frontend types should be generated from this, not hand‑written.

### 6. Gotchas the repo can't tell you on first glance

- **Race detector locally:** no gcc on Windows → `make test` works in CI but
  may fail to compile locally with `-race`. Use `go test ./...` (no `-race`)
  for local runs; CI runs `-race` on every PR via `.github/workflows/test.yml`.
- **Don't commit binaries:** `tmp/`, `backend/tmp/`, and `*.exe` are
  gitignored. Two `.exe` files (`backend/api.exe`, `backend/replay_dbn.exe`)
  are currently sitting in the worktree — leave them alone, do not stage them.
- **Don't commit data:** `backend/data/` is the 2.4 GB DBN archive (211M ticks,
  9 trading days). Gitignored. Never include it in commits.
- **`.kilo/state/`** is gitignored — chat transcripts and per‑session config
  don't belong in git.
- **`POSTGRES_*` env required for migrations.** `make migrate-up` interpolates
  `$POSTGRES_USER`, `$POSTGRES_PASSWORD`, `$POSTGRES_HOST`, `$POSTGRES_PORT`,
  `$POSTGRES_DB`. Source `.env` first or migrations will silently target the
  wrong DB.
- **DBN replay:** `dbn-go` v0.9.1 cannot decode the v1 InstrumentDef Databento
  served. Workaround is `backend/scripts/dbn_to_postgres.py` (Python bridge,
  has its own venv at `backend/scripts/validation/.venv/`).
- **Three deferred replay bugs (math output is currently zero):** futures
  contract reconstruction, wall‑clock vs event‑time TTE, and NATS 1 MiB
  payload cap. Detailed in `backend/HANDOFF.md` and `HANDOFF.md`.
- **Hard blocker (vendor):** Databento OPRA account locked. Live verification
  + DPI/Charm/Pin calibration cannot proceed until unlock. Offline calibration
  is unblocked once the three replay bugs are fixed.
- **PROGRESS.md is the cross‑session journal.** Update `backend/docs/PROGRESS.md`
  after meaningful backend work — that's how the next session catches up.
- **Existing per‑agent assets:** `.claude/agents/*.md` (Claude Code subagents)
  and `.claude/skills/*` exist and are still useful as reference for what
  specialized work has been split out (greeks validator, hot‑path reviewer,
  web design guardian, etc.). Don't move or delete them.
